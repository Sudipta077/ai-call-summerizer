# Project Instructions for Claude

## Modification Policy

When asked to modify code, add new features, or write new code in this project:

- **Do NOT change any existing functionality.** Only add or modify what is explicitly requested.
- If implementing the request requires changing existing behavior, logic, function signatures, imports, or any other working code — **always ask the user for confirmation BEFORE making that change.**
- Describe clearly what needs to change and why, and wait for the user's approval before proceeding.
- Preserve existing variable names, log statements, formatting, and code style unless the user explicitly asks otherwise.
- Do not refactor, rename, or "clean up" unrelated code as a side effect of a task.
